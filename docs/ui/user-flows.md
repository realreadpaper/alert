# 用户流程与状态细节

## 1. 首次启动

目标：让用户在 30 秒内理解产品，并进入创建或加入流程。

流程：

1. 展示产品定位页。
2. 用户选择“创建看护组”或“加入看护组”。
3. 申请必要权限。
4. 进入对应流程。

首屏文案：

```text
本地看护你的多台手机
不需要账号，不上传位置。设备离开时，组内手机会提醒你。
```

按钮：

- `创建看护组`
- `加入看护组`

## 2. 创建看护组

流程：

1. 用户点击 `创建看护组`。
2. App 生成本机 `deviceId` 和 `groupSecret`。
3. 用户输入或确认本机设备名。
4. App 创建默认看护组。
5. 进入首页，提示可添加设备。

设备名默认值：

- `主力 iPhone`
- `商务 Android`
- `备用机`

如果无法判断用途，则使用机型名。

## 3. 添加设备

流程：

1. 首页点击 `添加设备`。
2. 生成一次性邀请。
3. 展示二维码和有效期。
4. 新手机扫码。
5. 当前手机收到加入请求。
6. 用户点击 `允许`。
7. 两端保存组信息。
8. 首页设备列表新增设备。

失败状态：

- 邀请过期：`邀请已过期，请重新生成`
- 二维码无效：`无法识别此邀请`
- 创建者拒绝：`加入请求已被拒绝`
- 密钥交换失败：`加入失败，请重新邀请`

## 4. 加入看护组

流程：

1. 新手机点击 `加入看护组`。
2. 扫描二维码。
3. 确认设备名称。
4. 显示等待确认页。
5. 创建者允许。
6. 新手机进入首页。

等待确认页必须有取消入口：

- `取消加入`

超时后：

- `邀请已过期，请让对方重新生成`

## 5. 开始看护

流程：

1. 用户在首页点击 `开始看护`。
2. App 检查权限。
3. 权限完整：进入看护中。
4. 权限缺失：进入权限修复页。
5. 看护开始后，首页显示当前模式和设备状态。

权限修复页只显示与当前问题相关的项，不一次性列出所有系统权限。

示例：

```text
需要开启蓝牙
用于发现看护组内手机是否仍在身边。
```

按钮：

- `去开启`
- `稍后`

## 6. 模式切换

首页提供三个模式：

- 外出
- 室内
- 静音

点击后立即生效，不打开复杂设置。

模式说明可在首次点击时用轻提示展示：

- 外出：`适合出门和商务场合，提醒更及时。`
- 室内：`适合办公室和家中，减少误报。`
- 静音：`只通知和震动，不主动响铃。`

## 7. 预警流程

触发条件：

- 某设备持续失联达到当前模式预警阈值。

体验：

1. 其他设备震动。
2. 发送本地通知。
3. 首页状态变为“需要注意”。
4. 点击通知进入预警页。

预警页操作：

- `我知道了`：停止本次提醒。
- `暂停 10 分钟`：暂时不再提醒该设备。

如果设备恢复：

- 自动回到安全状态。
- 不弹出成功提醒，避免打扰。

## 8. 强提醒流程

触发条件：

- 预警后仍持续失联。
- 当前模式允许响铃。

体验：

1. 播放本地报警音。
2. 震动。
3. 显示报警页。
4. 通知保留停止入口。

停止条件：

- 用户点击 `我知道了`。
- 用户点击 `暂停 10 分钟`。
- 设备恢复连接。
- 用户切换静音模式。

## 9. 设备恢复

设备恢复连接后：

- 状态从 `preAlert` 或 `alarming` 回到 `online`。
- 首页显示安全。
- 如果正在响铃，立即停止。
- 不生成历史位置或轨迹。

短时间反复断连：

- GuardEngine 应合并事件，不连续弹出多个报警。
- UI 可显示 `连接不稳定`，避免反复强提醒。

## 10. 停止看护

用户点击 `停止看护` 后：

- 本机停止主动报警。
- 本机可继续显示组状态，但标记为暂停。
- 组内其他设备看到该设备 `已暂停`，不按丢失处理。

如果停止看护会影响他人，应提示：

```text
停止后，此手机不会再提醒你其他设备离开。
```

按钮：

- `继续停止`
- `取消`

## 11. 移除设备

流程：

1. 设置页进入设备详情。
2. 点击 `移除此设备`。
3. 二次确认。
4. 本机更新设备列表。
5. 组内其他设备通过本地同步或下一次配对更新。

由于无云端，设备移除同步不是强实时。首版可以要求用户在所有设备附近时完成管理操作。

## 12. 清除本地数据

设置页提供 `清除本地数据`。

说明：

```text
这会退出当前看护组，并删除本机保存的组密钥和设备列表。
```

二次确认必须清楚，避免误删。
